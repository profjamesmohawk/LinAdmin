Technical Note: Course VM on Mohawk Residence Network

Introduction

The course VM will not work correctly when run from a host on the Mohawk Residence WIFI network. The symptom is that w01 will connect to s01 and the open Internet, but will not be able to resolve domain names. w01 will be able to ping servers like 8.8.8.8 but will not be able to resolve domain names like www.mohawkcollege.ca.

It appears that the residence DNS server will not answer queries from the VM, so we need have the VM use alternate DNS servers.

The VirtualBox NAT interface passes the DNS server used by the host to the client via DHCP. The work around is to set the DNS server used by your host to point to DNS servers available on the open Internet.

Work Around:

Note: These steps apply to a Microsoft Windows host system. If you are using a Mac or Linux system, please contact your instructor.

From your VirtualBox host (the system running VirtualBox)

  1. From the Control Panel select 'Network and Internet'
  2. Select 'Network and Sharing Center'
  3. Select 'Change adapter settings'
  4. Right click on your adapter and select 'Properties', you should see something like...

  5. Double click on 'Internet Protocol Version 4 (TCP/IPv4)', and set the DNS server as indicated below...
