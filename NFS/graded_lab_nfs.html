Graded Lab: NFS

Introduction

Today you will demonstrate you mastery of NFS.

Grading:

You will run the host_info_nfs script on s01 and submit the output to Canvas. (host_info_nfs.zip)

Part A: Setup the lab

  1. Import the appliance which contains w01 and s01.
  2. Install the NFS utilities on all systems
  3. Make sure NFS is set to start automatically on s01
  4. Create users on w01 and s01 making sure they have consistent UID numbers.
    Full Name Employee Number Department User Name
    Margaret Hamilton 2000 research margaret
    Katherine Johnson 2001 research katherine
    Note: Running useradd on w01 will likely result in warnings from sss_cache. You can safely ignore these warnings. You can stop the warnings by running rm /var/lib/sss/db/* on w01.
  5. Create an group named research containing margaret and katherine on both systems.

Part B: Create "scratch" share

Create and export a directory named /nfs_shares/scratch for use as a temporary directory by any and all users on w01

  • The share is exported from s01
  • All users, including root should have read and write access
  • root on w01 should have root level access to the share

Test your work by mounting /nfs_shares/scratch on w01 and having root, margaret, and katherine each create a file.

Part C: Create a research share

Create and export a directory named /nfs_shares/research that is accessible only to the members of the research group on w01.

  • The share is exported from s01
  • Neither root or alice on w01 should be able to access the share
  • Both margaret and katherine must be able to create and delete files on the share
  • When members of the research group create files on the share, the files group ownership must be set to research automatically. hint: google "setgid bit on directory"

Test your work by mounting /nfs_shares/research on w01 and having margaret, and katherine each create a file. Verify that neither alice or root can access the files from w01.

Part D: Create A Public Share

Create and export a directory named /nfs_shares/pub from s01 as a public playground share. This one is a little different than what you've seen so far, some research will be required.

  • All users, including root, on w01 must have full access to the share
  • All users must appear as the user w01_guest when performing operations on the share.

Part E: Evaluation

  1. The script needs the netstat command to check for an active NFS connection. It is contained in the net-tools package. 
    yum install net-tools
Run host_info_nfs.zip on s01 and submit the output to the Canvas dropbox for this lab.
